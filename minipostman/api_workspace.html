📘 Dokumentasi API Alumni

              

                Dokumentasi penggunaan API Data Warehouse Serapan Kerja Alumni.
                Endpoint utama yang digunakan:
              

              /data_warehouse/backend/alumni.php
            


            

              
1. Alur Penggunaan Cepat

              
    
                
  • Login terlebih dahulu untuk mendapatkan api_key.
    • 
                
  • Copy nilai api_key dari response login.
    • 
                
  • Buka tab Auth, pilih X-API-KEY.
    • 
                
  • Paste API key, lalu uji endpoint data.
    • 
              

            


            

              
2. Auth Header

              

                Endpoint data seperti alumni, dashboard, charts, serapan, dan
                dimensi wajib memakai API key.
              

              X-API-KEY: api_key_dari_login
            


            

              
3. Register User

              POST
              /data_warehouse/backend/alumni.php?resource=auth&action=register
              { "username": "admin", "password": "admin123" }
            


            

              
4. Login & Ambil API Key

              POST
              /data_warehouse/backend/alumni.php?resource=auth&action=login
              { "username": "admin", "password": "admin123" }
              

                Jika berhasil, response akan mengembalikan data user dan
                api_key. API key itulah yang dipakai pada tab Auth.
              

            


            

              
5. Validasi API Key

              POST
              /data_warehouse/backend/alumni.php?resource=auth&action=validate
              

                Gunakan endpoint ini untuk mengecek apakah API key masih valid.
              

            


            

              
6. Regenerate API Key

              POST
              /data_warehouse/backend/alumni.php?resource=auth&action=regenerate_key
              
Membuat API key baru. API key lama otomatis tidak berlaku.

            


            

              
7. Logout

              POST
              /data_warehouse/backend/alumni.php?resource=auth&action=logout
              
Menghapus API key dari user yang sedang aktif.

            


            

              
8. Ambil Data Alumni

              GET
              /data_warehouse/backend/alumni.php?resource=alumni&page=1&limit=10
              

                Endpoint ini mengambil data alumni dengan pagination. Maksimal
                limit adalah 100 data per halaman.
              

              
    
                
  • page: nomor halaman.
    • 
                
  • limit: jumlah data per halaman.
    • 
                
  • tahun: filter tahun lulus.
    • 
                
  • prodi: filter prodi, contoh TKJ/TMJ.
    • 
                
  • status: filter status kerja.
    • 
              

              /data_warehouse/backend/alumni.php?resource=alumni&page=1&limit=100&prodi=TKJ&tahun=2024
            


            

              
9. Detail Alumni by ID Fact

              GET
              /data_warehouse/backend/alumni.php?resource=alumni&id_fact=1
              

                Digunakan untuk mengambil satu data alumni. Cocok untuk fitur
                cari data sebelum edit.
              

            


            

              
10. Tambah Data Alumni

              POST
              /data_warehouse/backend/alumni.php?resource=alumni
              

                Gunakan body mode raw JSON. Field wajib: kode responden,
                tahun lulus, prodi, jenjang, status kerja, kategori instansi,
                range pendapatan, lama tunggu, dan kota.
              

            


            

              
11. Update Data Alumni

              PUT
              /data_warehouse/backend/alumni.php?resource=alumni
              

                Untuk update, field id_fact wajib dikirim di body JSON.
              

            


            

              
12. Import Excel ETL

              POST
              /data_warehouse/backend/alumni.php?resource=alumni&action=import_excel
              

                Endpoint ini menggunakan upload file Excel/CSV dengan
                form-data key file. Di MiniPostman ini sudah
                tersedia Body Mode form-data / upload file untuk mengetes
                import langsung dari halaman ini.
              

            


            

              
13. Dashboard & Charts

              GET
              /data_warehouse/backend/alumni.php?resource=dashboard
              /data_warehouse/backend/alumni.php?resource=charts
            


            

              
14. Serapan Kerja

              GET
              /data_warehouse/backend/alumni.php?resource=serapan&action=summary
              /data_warehouse/backend/alumni.php?resource=serapan&action=prodi
              /data_warehouse/backend/alumni.php?resource=serapan&action=tahun
            


            

              
15. Data Dimensi

              GET
              /data_warehouse/backend/alumni.php?resource=dimensi&name=prodi
              /data_warehouse/backend/alumni.php?resource=dimensi&name=tahun
              /data_warehouse/backend/alumni.php?resource=dimensi&name=status
              /data_warehouse/backend/alumni.php?resource=dimensi&name=kota
              /data_warehouse/backend/alumni.php?resource=dimensi&name=instansi
              /data_warehouse/backend/alumni.php?resource=dimensi&name=pendapatan
            


            

              
16. Error Umum

              
    
                
  • 
                  400: request/body tidak lengkap atau format salah.
                
    • 
                
  • 
                  401: API key kosong, salah, atau sudah tidak valid.
                
    • 
                
  • 404: resource atau data tidak ditemukan.
    • 
                
  • 405: method tidak didukung oleh endpoint.
    • 
                
  • 409: data sudah ada atau duplikat.
    • 
                
  • 500: error server/database.
    •
